Create Builds

Create Enemy On Board builds here. You can choose a character, starting item, and perks. Information is displayed about the items and perks, so you can see the pros and cons of using them. Once you are done customizing, you can share them with your friends! They will more easily be able to grasp your strategies.

GitHub
API

The API can be queried for item and perk data. It contains many attributes such as damage and slow for each one of them. Perks have different attributes for each level. You can only make one request for information on either an item or a perk in each call.

Resources

Each attribute has a two elements: the first one is the value associated with it and the second one is the type of the value (either fixed, percent, str, or list).

400 Bad Request

If you request an character, item, or perk that is not on the lists here, you will get a 400 status error as the API cannot fetch for that value. The following is an example of what will be returned: 

{
    "message": "Character not found."
}

Rate Limiting

Rate limits are assigned on an IP address basis. You can be rate limited if you make too many requests in a period of time. Currently, there is a maximum of 1000 requests per 10 minutes. If you exceed this, you will receive information that will help in accounting for rate limits. The following will be returned: 

{
    "message": "You are being rate limited.", 
    "retry-after": 101.23
}

retry-after is the amount of time in seconds until another request can be made.

Headers

Headers will be sent will be sent with all responses and can be read to view information on rate limiting. The headers in the response will look like the following: 

< HTTP/1.1 429 TOO MANY REQUESTS
< Content-Type: application/json
< Retry-After: 102
< X-RateLimit-Limit: 10
< X-RateLimit-Remaining: 0
< X-RateLimit-Reset: 1607058918.26322
< X-RateLimit-Reset-After: 101.23
{
  "message": "You are being rate limited.",
  "retry_after": 101.23
}

Retry-After is only sent with rate limits, but the other headers are sent with every request:

  • Retry-After: The amount of time in seconds in which another request can be made.
  • X-RateLimit-Limit: The maximum number of requests that can be made.
  • X-RateLimit-Remaining: The number of remaining requests that can be made.
  • X-RateLimit-Reset: The epoch time in which the rate limit resets.
  • X-RateLimit-Reset-After: The amount of time in seconds in which the current rate limit will be reset. It can fluctuate in decimals due to millisecond precision.
